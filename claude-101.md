## 什麼是 Claude？

（10 分鐘）

Claude 不只是一個聊天機器人——它是一個 AI 助手，設計來成為你的思考夥伴。在這一課中，你將學習是什麼讓 Claude 有別於其他 AI 工具，並了解它如何在各種工作任務中提供協助。

### 了解 Claude 的能力

Claude 能協助的任務範圍遠超過簡單的問答互動，發展成為一種既能自動化又能增強你工作的助理夥伴關係。以下是 Claude 擅長的幾個面向：

- 寫作與內容創作：Claude 能與你在社群媒體貼文、專業電子郵件和複雜報告上合作。由於 Claude 在個性與語氣上受過指導訓練，你可以一起反覆調整結構和清晰度，直到你的聲音清楚傳達出來。
- 研究與分析：Claude 幫助你探索研究角度、整理發現並分析資料以發掘有意義的洞見。你可以上傳文件，Claude 將幫助你理解複雜的資訊——這得益於 Claude 龐大的 context window，可在單次對話中處理 200K+ 個 token（約 500 頁文字或更多）；Pro、Max、Team 和 Enterprise 方案使用 Opus 4.7 時最多可達 100 萬個 token。
- 程式碼協助：Claude Opus 4.7 是我們迄今最強大的模型，也是世界上最好的程式碼編寫模型。在真實世界程式碼任務上的卓越表現意味著 Claude 能幫助你用多種程式語言撰寫、除錯和解釋程式碼。
- 問題解決與推理：Claude 處理複雜的認知任務、數學問題、策略性思考與分析，以及研究工作。Claude Opus 4.7 和 Sonnet 4.7 是混合模型，提供兩種模式：近即時回應與用於更深層推理的延伸思考（extended thinking）。延伸思考讓 Claude 能逐步解決問題，非常適合需要仔細分析的任務。
- 學習新事物：無論你是在學習新技能、探索陌生領域，還是解決複雜挑戰，Claude 都能適應你的學習風格和節奏。學習模式是一種全新的 Claude 體驗，它引導你的推理過程而非直接提供答案，有助於培養批判性思維能力。

前往 claude.com 的使用案例庫，找尋適合你特定職能的使用靈感。若想深入了解 AI 能做什麼和不能做什麼，請參閱我們的 AI 能力課程。

### 存取 Claude 的方式

Claude 是那個智能——你在本課程中學習與之合作的 AI 助手。同樣的智能可以透過多種介面使用，各自適合不同類型的任務。

- Claude.ai（以及相關的行動與桌面應用程式）是大多數人與 Claude 互動的主要方式。在這裡，你可以提問、腦力激盪、創作和編輯文件等。Claude.ai 非常適合對話、寫作協助、研究、分析和建立檔案。這也是本課程的重點。
- Claude Code 是一個代理型程式碼工具，為開發者設計，但也可以用於桌面上各種檔案操作。Claude Code 可以直接編輯檔案、執行命令並建立 commits。
- Claude 與 Slack 將 Claude 直接帶入你團隊的溝通工具。你可以在任何頻道或對話的 AI 助手標題中與 Claude 聊天，或在串流中 @Claude。連接 Slack 後，Claude 會搜尋你工作區的頻道、私訊和共享檔案，為你提供更好的回應和研究所需的背景。
- Claude for Excel 讓你可以在 Microsoft Excel 的側欄直接與 Claude 合作，Claude 能讀取、分析、修改和建立新的 Excel 工作簿。Claude for Excel 最適合模型分析、假設更新、錯誤除錯、範本填充、公式說明和多頁簽導航。

本課程主要聚焦於 Claude.ai，但你也可以查看《Claude Code in Action》以了解更多在開發工作流程中使用 Claude 的資訊。

## 與 Claude 的第一次對話

### 學習目標

完成本課後，你將能夠：

- 開始與 Claude 的新對話並導航介面
- 使用清晰、具體的語言撰寫有效的提示
- 上傳檔案和圖片為 Claude 提供額外背景
- 使用後續訊息反覆迭代和精煉 Claude 的回應

### 開始你的第一次對話

開啟 Claude.ai 時，你會看到一個簡潔的介面，螢幕底部有一個文字輸入區域。

你的提示可以從簡單的問題（如為新功能腦力激盪代號名稱）到複雜的共同建立檔案請求。

### 撰寫有效的提示

所有與 Claude 的互動都從提示開始，這些提示結合其他背景資訊影響 Claude 的回應。與 Claude 說話的最佳方式是像與同事說話一樣——自然、簡潔、以對話方式進行。

在下次與 Claude 對話前，考慮以下幾點：

- 設定情境（Setting the stage）：你的角色是什麼，你的目標是什麼？有沒有關於你工作的背景是 Claude 應該知道的？
- 定義任務（Defining the task）：你希望 Claude 採取什麼行動？你希望 Claude 寫作、分析、建立，還是其他？
- 指定規則（Specifying rules）：你希望 Claude 使用什麼風格或語氣？你可以附上範例來展示你在尋找什麼嗎？

#### 整合運用

以下是一個使用了三個元素的範例提示：

> 「我是一家獨立串流新創公司的行銷主管，我們正在為 A 輪投資者準備投資者簡報。你能研究一下獨立電影串流市場的現狀，並識別關鍵趨勢、競爭對手定位和成長機會嗎？使用帶有引用的最新網路研究，並將其結構化為最多 5 頁的專業報告，包含執行摘要、市場分析、競爭格局和成長機會。」

在這個提示中：

- 設定情境：我們告訴 Claude 這是為一個新獨立串流應用程式的投資者簡報——這是背景和目標。
- 定義任務：我們提供了具體的行動（研究市場）以及相關細節（趨勢、競爭對手、機會）。
- 指定規則：我們要求帶有引用的最新網路研究，結構化為專業報告——精確告訴 Claude 我們需要的風格和格式。

> 想要深入了解嗎？這個提示框架改編自 AI 應用能力的 4D 框架，由 Rick Dakan 教授（林林藝術設計學院）和 Joseph Feller 教授（科克大學學院）透過研究合作開發。該框架識別出四種核心能力——Delegation、Description、Discernment 和 Diligence——讓你能有效地與 AI 協作。
>
> 參加我們的免費 AI Fluency 課程，了解這四種能力如何共同實現高效、有效、合乎倫理且安全的 AI 使用。

### 新增背景

上傳、connectors 和自訂偏好設定提供了為 Claude 提供更多工作背景的方式。

Claude 可以分析 PDF 和其他文件中的文字和視覺元素（如圖片、圖表和圖形）。支援的檔案類型包括 PDF、DOCX、CSV、TXT，以及 PNG 和 JPEG 等常見圖片格式。

實際使用檔案上傳的幾種方式：

- 上傳文件並請 Claude 摘要要點
- 分享圖片並請 Claude 描述或分析它所看到的內容
- 附上試算表並請 Claude 識別資料中的趨勢
- 上傳程式碼並請 Claude 解釋其運作原理或找出錯誤

上傳後，Claude 將自動嘗試解析檔案內容。在聊天中，檔案以附件形式出現，然後你可以向 Claude 提問。

> 小技巧：如果你希望 Claude 在每次回應中都考慮特定偏好，前往「Settings > General > 'What personal preferences should Claude consider?'」設定適用於每次對話的偏好。

### 迭代 Claude 的回應

與 Claude 的對話本應是迭代的。將一口大小的提示串連起來，形成一段自然的對話，讓你根據 Claude 的回覆引導對話走向。

如果 Claude 的第一個回應不太符合你的期望，你有幾個選擇：

- 提出後續問題：在 Claude 的回應基礎上進一步詢問更多細節、不同角度或澄清。例如：「能詳細說明第二點嗎？」或「這很有幫助，但能不能更簡潔一點？」
- 提供回饋：告訴 Claude 你喜歡和不喜歡它的回應的什麼。「這很好，但語氣太正式了。能讓它更有對話感嗎？」
- 重新導向或重新開始：如果 Claude 走向了你沒有預期的方向，直接把它拉回來。「其實，我問的是 X，不是 Y。讓我澄清一下……」最壞的情況，在新的聊天中重新開始，完全刷新背景。

> 小技巧：你也可以點擊任何訊息上的鉛筆圖示來編輯並重新提交你的提示——當你想精煉請求而不是添加新訊息時非常有用。

### 個性化 Claude

有兩個功能可以幫助 Claude 隨著時間為你工作得更好，增強你提示的效果。

Memory（記憶）自動從你的對話中儲存重要背景——你的角色、偏好、過去的決定和工作風格——這樣你就不必在每次開始新聊天時重複這些。例如，如果你告訴 Claude 你在一家 B2B 公司做行銷，它會記住那個背景。你隨時可以在「設定」中查看、編輯或刪除 Claude 記住的任何內容，而且記憶在你所有裝置上同步。

Styles（風格）讓你自訂 Claude 的溝通方式。從預設選項中選擇——如簡潔、正式或解釋型——或通過描述你希望 Claude 如何寫作來建立你自己的自訂風格。設定後，你的風格會自動應用於所有對話。

## 獲得更好的結果

### 學習目標

完成本課後，你將能夠：

- 認識開始使用 AI 時的常見挑戰，並使用疑難排解技巧來克服它們

- 說明你可能如何設定評估（evals）來更好地了解 Claude 在你獨特工作流程中的表現

### 常見挑戰及解決方式

開始使用 Claude 時，你可能會遇到回應不符合預期的時刻。這很正常——這是精煉方法的機會。以下是一些最常見的挑戰以及如何解決它們。

| 挑戰 | 發生的原因 | 嘗試這個 |
|------|----------|---------|
| Claude 的回應太籠統 | 你的提示沒有包含足夠關於你具體情況的背景 | 加入關於你的受眾、角色或限制的細節。不要只說「寫一封關於專案延遲的電子郵件」，試試「寫一封電子郵件給我們的企業客戶，解釋軟體整合將延遲兩週。他們目前一直很有耐心，但這是第二次延遲了。保持專業但要道歉。」 |
| 回應太長（或太短） | Claude 在猜測適當的長度 | 明確說明：「給我一個兩段落的摘要」或「保持在 100 字以內」或「我需要一個全面的分析——長度不是問題。」 |
| Claude 沒有遵循我的格式 | Claude 理解你想要什麼，但不知道你希望如何呈現 | 展示而不只是告訴。提供格式範例，或明確描述結構：「每個部分使用帶有粗體標題的條列式清單。」 |
| 我得到了聽起來很自信但事後發現是錯誤的資訊 | Claude 偶爾會生成合理但不正確的資訊，尤其是針對具體事實或小眾主題 | 對於高風險工作，獨立驗證關鍵事實。請 Claude 引用來源或指出信心水準。啟用網路搜尋以將回應建立在最新資訊上。 |
| 語氣不對 | Claude 預設為有幫助且專業，這可能不符合你的需求 | 用平白的語言描述語氣：「讓這個更有對話感」或「這應該聽起來權威且正式。」提供一個你想要的寫作風格範例。 |

### 迭代思維

使用 Claude 時最重要的轉變之一是認識到你的第一個提示很少能產生完美的結果——這沒關係。將你的初始提示視為對話的開始，而不是一次性的請求。

有效使用 Claude 的用戶：

- 將第一稿視為起點。審閱 Claude 產生的內容，識別什麼有效、什麼無效，然後改進。
- 給予具體回饋。「讓它更短」很好，但「刪掉前兩段，讓結論更有行動導向」更好。
- 知道何時重新開始。如果對話已經偏離軌道，有時候以更清晰的提示開啟新的聊天比試圖重新導向更快。

### 什麼是 AI 應用能力（AI Fluency）？

AI 應用能力是與 AI 工具有效協作的能力——不只是知道按哪個按鈕，而是培養在不同情境中善用 AI 的判斷力。

AI 應用能力的 4D 框架，由 Rick Dakan 教授（林林藝術設計學院）和 Joseph Feller 教授（科克大學學院）透過研究合作開發，識別出四種核心能力，共同幫助你充分發揮 AI 互動的潛力：

- Delegation（授權）：決定哪些工作應由人類完成、哪些由 AI 完成，以及如何分配任務。包括理解你的目標、AI 的能力，以及做出策略性的協作選擇。
- Description（描述）：與 AI 系統有效溝通。包括清楚定義輸出、引導 AI 流程，以及指定期望的 AI 行為和互動方式。
- Discernment（識別）：深思熟慮且批判性地評估 AI 的輸出、流程、行為和互動。包括評估品質、準確性、適當性，以及確定改進領域。
- Diligence（盡責）：負責任且合乎倫理地使用 AI。包括對 AI 系統和互動做出深思熟慮的選擇、保持透明度，以及對 AI 輔助的工作負責。

你在這整個課程中一直在練習這些技能。第二課的提示框架（設定情境、定義任務、指定規則）根植於 Description。上面的疑難排解技巧借鑒了 Discernment 和 Diligence。

若要了解更多，請參加我們深入探索所有四種能力的免費 AI Fluency 課程，包含實際練習和真實世界應用。

### 為你的工作流程評估 Claude

當你開始將 Claude 整合至更多工作中，你可能會想：我如何知道 Claude 在這個特定任務上是否真的表現良好？

這就是 Discernment 變得至關重要的地方。評估（Evals，即 evaluations 的縮寫）是一種培養直覺以評估 Claude 在對你重要的任務上輸出品質的方式。它們是系統性地測試 Claude 在對你重要的特定類型任務上表現如何的方法。

#### 為什麼評估很重要

你的工作是獨特的。Claude 可能在起草行銷文案上表現出色，但在你特定領域的技術文件上需要更多指引。進行簡單的評估有助於你：

- 了解 Claude 在你工作流程中哪裡最有價值
- 識別你需要提供更多背景或範例的任務
- 建立對 Claude 重複性任務輸出的信心

#### 一個簡單的評估方法

你不需要複雜的基礎設施來評估 Claude。以下是一個實際可行的方法：

1. 收集範例。收集你定期進行的 5–10 個任務範例——你寫過的電子郵件、你建立的報告、你完成的分析。
2. 建立測試提示。撰寫能產生類似輸出的提示。包含你在進行這項工作時自然會有的背景。
3. 比較輸出。執行你的提示並比較 Claude 的回應與你的範例。問自己：
   - Claude 有沒有掌握關鍵資訊？
   - 語氣和風格是否適當？
   - 什麼缺失了或可以改進？
4. 精煉你的方法。根據你的學習，調整你的提示、加入範例來展示好的結果是什麼樣子，或識別人工審閱至關重要的地方。

#### 範例：使用 Claude 進行資料分析

要評估 Claude 如何使用你的資料：

1. 找到一個你手動分析過的資料集
2. 建立提示，要求 Claude 代表你進行分析
3. 比較 Claude 的結果與你的原始結果
4. 記錄模式並相應精煉你的提示：也許 Claude 能獲得正確的數字，但錯過了整體模式

這種輕量級評估幫助你為對你重要的任務培養直覺——以及在哪裡聚焦你的審閱和精煉精力。

## Claude 桌面應用程式：Chat、Cowork、Code

### 學習目標

### 導航 Claude 桌面應用程式

Claude 桌面應用程式提供三種與 Claude 合作的方式：Chat、Cowork 和 Code——從快速提問到複雜研究再到建立軟體。

- Chat 與你在 claude.ai 上了解的 Claude 相同，加上快速輸入、截圖、口述，以及因在你的電腦上原生執行而來的 connectors。
- Cowork 是一個代理型工具——你給它一個目標，連接你的工具和資源，讓它去完成工作。透過 Cowork，Claude 有足夠的觸及範圍和空間做更多事。這更廣泛的範疇讓它能進行更徹底的研究與分析，並產出更複雜的文件和成果。
- Code 用於建立軟體，從撰寫和測試程式碼到部署。

Cowork 和 Code 執行在同一個引擎上。兩者底層都是 Claude Code——本地執行在你的電腦上，能夠獨立工作，能啟動子代理並維持長期任務。這讓 Claude 能自行完成較大的任務，如研究和寫作或建立軟體。

每種模式都圍繞它所服務的工作設計，顯示重要的內容，並在你需要的地方給你控制。

### Chat

Chat 最適合需要提問、腦力激盪、起草或反覆解決問題的情況。

如果你用過 claude.ai，使用方式相同，加上幾個因原生執行在你電腦上而帶來的功能：

- 快速輸入：在 Mac 上雙擊 Option 鍵，在你正在做的事情上方呼出 Claude。它在一個保持置頂的緊湊視窗中回應，當你在應用程式之間切換時也保持顯示。你永遠不必離開正在做的事情來提問。
- 截圖和視窗分享：擷取截圖或分享視窗，讓 Claude 精確看到你正在看的內容。比描述螢幕上的內容更快，也更精確。（Mac）
- 口述：通過說話來思考問題，而不是打字。當你在大聲思考、遠離鍵盤，或在說話比寫作更快的情況下進行思考時很有用。（Mac）
- 桌面 connectors：透過 connectors 連接本地工具和服務，讓 Claude 能使用你電腦上的其他工具。

適合使用的時機：

- 當你盯著一個陌生的儀表板時。雙擊 Option，將游標拖過視窗截圖，然後問「這些指標是什麼意思？」Claude 在疊加視窗中回答，而儀表板保持在視野中。
- 當你在會議之間想思考如何構建一個簡報的結構時。開啟快速輸入，切換到語音，然後說出來。Claude 從你說的內容起草一個大綱。
- 如果你幾週來一直在 Apple Notes 中記錄產品發布的想法。從「設定」新增 Notes connector，然後問 Claude：「整理我所有關於 Osprey 發布的筆記，找出我哪裡沒有完成，並查看我其他連接的工具，看看有沒有填補空白的內容。」Claude 讀取你電腦上的筆記，整理你所擁有的，並在你停下的地方跟進。

### Cowork

Claude Cowork 是為需要真正努力的工作而建的：從多個來源提取資訊、理解它，並產出完成品。

在 Cowork 中，Claude 可以多工處理，同時處理專案的不同部分，因此它有足夠的範疇從更多來源汲取，並有毅力看事情到底。徹底的研究簡報、跨來源的財務分析、端到端的合約審查、從分散在多個來源的材料中整理出精美的幻燈片。

開始前，Claude 通常會問一組簡短的問題來確定你的需求：範疇、格式、限制。它在側欄中建立一個你可以審閱的計畫。工作進行時，你可以看到任務逐漸成形：它從哪些來源汲取、正在形成的檔案、計畫的進度。你可以同時執行多個任務，每個任務都在自己的對話中，並從側欄切換。

Cowork 的主要功能：

- 資料夾存取：讓 Claude 指向你電腦上的一個資料夾，它讀取那裡的內容，找出相關的，並將完成的工作儲存回同一個地方。你也可以上傳檔案、將內容貼入對話，或連接提取 Claude 所需內容的工具。
- 排程任務：Claude 可以按排程處理重複性工作：每日提取 Slack 和行事曆的摘要、每週已完成工作的摘要、每日對需要你注意的內容進行收件匣分類。你定義任務和執行時間，每次應用程式開啟時 Claude 自動處理。如果你的電腦或應用程式在任務預定時間關閉，它會在你回來時補上。
- 子代理：Claude 啟動的後台工作者，用於平行處理任務的各個部分。如果你請求複雜的事情——比如從多個來源提取的研究簡報——Claude 將其分解為子任務，為每個子任務分配一個有自己背景的子代理，並協調結果，為你提供一份完成的成果。
- Dispatch：一個持久的對話串，讓你可以從手機繼續你的 Cowork 對話。從 Claude 行動應用程式，你可以交給 Claude 使用電腦上所有內容的任務——你的檔案、connectors、plugins，甚至桌面應用程式。使用此功能需要桌面和行動應用程式，且電腦必須處於喚醒狀態、桌面應用程式開啟。
- Projects：Cowork 中的 Projects 讓你將相關任務分組到具有自己檔案、背景、指令和記憶的專屬工作區中。如果你在 Claude 上使用 projects，Cowork projects 的運作方式類似，但它們本地存放在你的桌面上，並圍繞你透過 Cowork 執行的任務構建。
- 瀏覽器使用：在 Chrome 中連接 Claude，Claude 可以導航網站、與頁面互動，並將找到的內容直接提取到正在處理的任務中。這就是 Cowork 如何完成跨十個網站查看競爭對手定價或從沒有 API 的頁面收集資料等任務。
- 電腦使用：當 Claude 沒有你所需的 connector 或 plugin 時，它可以直接導航你的電腦——點擊、打字和開啟應用程式，就像你一樣。Claude 遵循優先順序：先是 connectors，然後是 Chrome，最後是螢幕互動，因此它總是選擇最快、最可靠的路徑。每次 Claude 存取某個應用程式前，你都會看到一個權限提示，你可以為任何你想排除的內容設置封鎖清單。電腦使用功能在 Pro 和 Max 方案的研究預覽中，僅限 macOS（Windows 即將推出）。
- Plugins：Plugins 給 Claude 它本身沒有的能力：提取即時財務資料、搜尋你公司的內部知識庫，或在特定合規框架內工作。從 Cowork 介面瀏覽並新增它們以適應任務。
- 受保護環境：Cowork 在你電腦上的一個受限空間中執行。Claude 可以在你分享的資料夾內讀取、建立和編輯檔案，但無法存取其外的任何內容。

適合使用的時機：

- 你想像查詢資料庫一樣查詢你所有的工具。詢問「回顧上季我們關於定價的決策，查看會議記錄、Slack 和電子郵件，然後用這些發現更新我們的 Q3 簡報？」Cowork 在會議記錄、幻燈片、電子郵件和 Slack 串流中找到答案。
- 你在研究新市場、評估競爭對手、評估工具。對於任何可能跨多個標籤且難以提取資訊的研究，Cowork 訪問網站、閱讀報告、提取定價，並提供帶有來源的結構化簡報，而你不需要開啟任何瀏覽器標籤。
- 你需要處理一個包含 50 多份文件的資料夾，包括合約、財務報告和會議記錄。你可以請 Cowork 找出與你計畫最相關的文件，並製作摘要備忘錄。Cowork 閱讀每一頁，跨整個文件集交叉參考，並找出只有閱讀所有文件才能出現的模式。像查閱五份一樣查閱五十份。
- 你每天早上都在做同樣的工作——查看訊息、整理狀態更新、為當天的會議做準備。設定一次作為排程任務，Claude 就會重複處理，讓你以答案而非行政工作開始一天。

Cowork 適用於 Pro、Max、Team 和 Enterprise 用戶，並定期新增新功能。

### Code

Code 標籤讓你能在桌面應用程式內直接使用 Claude Code 的強大功能，為你提供建立軟體的完整開發環境。

透過 Code，Claude 直接在你的程式碼庫中工作：讀取現有內容、撰寫和修改程式碼、執行命令。視覺化的 diff 顯示變更了什麼，內建終端機顯示命令執行情況，git 追蹤每個版本，讓你隨時可以回滾。

Cowork 在限定於你分享的資料夾的受限工作區中執行，而 Code 則直接在你的專案中執行，可完全存取你的檔案系統、終端機和開發工具。

你可以選擇工作發生的位置：

- 本地：你選擇電腦上的一個資料夾，Claude 直接使用這些檔案工作。因為它在你的機器上執行，Claude 可以讀取你的專案、存取本地工具，並執行你可以在瀏覽器中預覽的開發伺服器。
- 遠端：你連接一個 GitHub repository，Claude 在雲端環境中工作。即使你關閉應用程式，工作階段也會繼續，因此你可以開始一個大型重構並稍後回來查看。適合較大的程式碼庫，或當你希望將開發工作從本地機器移開時。

三種互動模式讓你控制 Claude 自行完成多少工作：

- Ask：Claude 提議每個變更並等待你的批准。你審閱視覺化的 diff，並在任何內容被修改前接受或拒絕。
- Code：Claude 自動應用檔案變更，但在執行終端機命令前先確認。
- Plan：Claude 在碰任何東西之前先概述其完整方法。一個專用的計畫檢視器讓你在工作進行時審閱和重新查看策略。

你可以跨專案執行多個工作階段，並在側欄中按狀態（Active 或 Archived）和環境（Local 或 Cloud）篩選。

Code 標籤適用於 Pro、Max、Team 和 Enterprise 用戶。

### 三種模式的比較

| | Chat | Cowork | Code |
|--|------|--------|------|
| 優化目標 | 較快的交流：探索想法、迭代起草、快速解答、透過對話學習 | 複雜或持久的工作：研究、分析、檔案整理、產出完成的文件和成果 | 建立軟體：撰寫、測試、執行和部署程式碼 |
| 主要功能 | 快速輸入、口述 | 從本地資料夾工作、plugins、子代理、排程任務 | Ask/Code/Plan 模式、視覺化 diff、git 整合、本地和遠端環境 |
| 工具與擴充 | Connectors、Skills、Claude in Chrome | Connectors（本地和遠端）、Skills、Claude in Chrome、Plugins、電腦使用 | Connectors、Skills、Claude in Chrome、Plugins、Hooks |

## Projects 入門

### 學習目標

完成本課後，你將能夠：

- 說明 Projects 是什麼以及何時使用它們
- 建立一個帶有名稱、描述和可見性設定的新 project
- 向你的 project 知識庫新增文件和檔案
- 撰寫有效的 project 指令來引導 Claude 的行為
- 與團隊成員分享 projects（適用於 Claude for Work（Team 和 Enterprise 方案）用戶）

### 什麼是 Projects？

Projects 非常適合儲存 Claude 應該參考的知識、將圍繞特定主題或工作領域的相關聊天組織起來，以及與需要存取相同共享背景的團隊成員協作。

#### 何時使用 Projects

當你在處理持續性的事情時——不只是一次性問題——Projects 特別有價值。考慮在以下情況建立一個 project：

- 你會反覆使用的參考資料（會議記錄、調查結果、報告、歷史資料等）
- Claude 回應方式的一致性要求（永遠使用正式語言、永遠引用來源、永遠遵循我們的範本）
- 多個人需要從相同基礎工作的團隊協作需求

### 建立你的第一個 Project

設定一個 project 只需幾分鐘。以下是入門方式：

#### 步驟一：設定你的 Project

1. 將游標懸停在左側側欄並點擊「Projects」，或直接導航至 claude.ai/projects
2. 點擊右上角的「+ New Project」
3. 給你的 project 一個描述性名稱（例如「Q4 Marketing Campaign」或「Product Documentation」）
4. 加入你正在處理的內容的簡短描述。雖然 Claude 不會直接看到這個描述，但它能幫助你和你的團隊成員了解 project 的目的。
5. 選擇你的可見性設定：保持私人或與你的組織分享（適用於 Claude for Work 用戶）

#### 步驟二：加入 Project 指令

Project 指令告訴 Claude 如何在這個 project 的所有對話中行為。點擊「Instructions」開啟指令面板。

好的 project 指令通常包括：

- 關於你正在處理的內容的背景：「這個 project 是為我們的 B2B 軟體產品建立行銷內容。」
- 流程指令：「首先考慮一個能吸引這個受眾的部落格結構，然後撰寫草稿。」
- 語氣和風格偏好：「使用專業但有對話感的語氣。盡可能避免術語。」
- 具體要求：「行銷文案的末尾永遠包含行動呼籲。」

撰寫指令後，點擊「Save instructions」。這些指令將適用於這個 project 中的每次聊天，並與你設定的任何用戶偏好和風格一起運作。

你也可以使用 project 指令來自動化工作流程——例如，「當我上傳會議記錄時，使用這個範本建立一個結構化的摘要。」將指令視為為這個 project 設定 Claude 行為的程式設計。

#### 步驟三：建立你的知識庫

你的 project 知識庫是你上傳 Claude 應該參考的文件的地方。你可以在 project 主頁右側找到檔案選單。

點擊「+」按鈕新增內容。你可以上傳各種檔案類型，包括 PDF、DOCX、CSV、TXT、HTML 等。你也可以連接 Google Drive 直接連結文件。

應該上傳什麼：

- 參考文件（品牌規範、風格指南、範本）
- 背景資料（研究報告、會議記錄、需求文件）
- 你希望 Claude 仿效的工作範例
- 技術文件或規格

> 小技巧：為你的檔案取描述性名稱。Claude 使用檔案名稱來理解和檢索正確的資訊，因此「Q4-2024-Brand-Guidelines.pdf」比「document1.pdf」更有幫助。

### Projects 如何處理大型知識庫

你可能想知道上傳大量內容時會發生什麼。Projects 透過一個叫做檢索增強生成（Retrieval Augmented Generation，RAG）的過程自動擴展以處理大量內容。簡而言之，這意味著 Claude 能自動找到並使用你上傳文件中最相關的部分來回答問題，而不需要你指定要看哪個檔案。

當你的 project 知識接近 context window 限制時，Claude 會無縫啟用 RAG 模式。Claude 不是一次將所有 project 內容載入記憶，而是智慧地搜尋並只檢索回答你問題所需的最相關資訊。這將你的 project 容量擴大了 10 倍，同時保持回應品質。

當你的 project 啟用了 RAG 時，你會看到視覺指示，但體驗應該感覺相同——你仍然可以上傳文件、與 Claude 聊天，並獲得有背景感的回應。

### 在你的 Project 中工作

設定好 project 後，你可以開始與 Claude 聊天。project 內的每次對話都自動能存取你的知識庫，並遵循你的 project 指令。

### 協作功能

對於 Claude for Work（Team 和 Enterprise）方案的用戶，projects 透過協作功能變得更加強大。

#### 權限等級

分享 project 時，你可以選擇三種權限等級：

| 權限 | 說明 |
|------|------|
| Can view | 成員可以看到 project 內容、存取知識庫並聊天——但不能做任何變更。可以想成是帶有討論權限的唯讀存取。 |
| Can edit | 成員擁有完整的協作能力。他們可以修改指令、更新知識庫、管理其他成員，並積極貢獻於 project。 |
| Owner | Project 建立者控制一切，包括誰能看到 project。他們可以與特定人員分享，或讓 projects 對整個組織可見。 |

#### 分享你的 Project

分享 project 的方式：

1. 開啟你想分享的 project
2. 點擊 project 名稱右方的「Share project」按鈕
3. 使用他們的名字或電子郵件新增個別成員，或複製貼上電子郵件地址清單以批量分享（在這種情況下，project 將出現在他們的「Shared with you」區段）
4. 或者，與「Everyone at [your organization]」分享，讓你的 project 在 Team 標籤中可被發現

當你與他們分享 project 時，團隊成員會收到電子郵件通知，他們可以在「Shared with me」標籤中找到共享的 projects。

### 激發靈感的 Project 範例

不確定從哪裡開始？以下是不同職能部門的一些常見 project 類型：

- Q4 產品發布：上傳你的產品規格、競爭分析和訊息腦力激盪筆記。Claude 在任何詢問或文件起草時都能把這些背景放在首位。
- 研究支援：整合你的競爭審查、用戶研究資料和客戶回饋。Claude 可以幫助你整合來源、起草報告，並在建議中保持一致性。
- 客戶帳戶中心：將客戶的品牌規範、過去的交付成果和溝通記錄保存在一個地方。設定指令，讓 Claude 在建立提案或報告時匹配他們的語氣並參考他們的具體背景。
- 活動規劃工作區：上傳場地合約、演講者簡介和與會者資料。Claude 可以幫助生成活動流程文件、與會者溝通和活動後報告，並與活動主題保持一致。
- 職位描述生成器：收集過去的職位描述、團隊章程和內部人員申請文件。與 Claude 合作起草反映你團隊實際工作和文化的職位描述。

### Projects 的最佳實踐

要充分利用 projects：

- 先聚焦，再擴展。從一個具體的使用案例開始，而不是試圖為所有事情建立一個 project。你隨時可以在進行中新增更多內容。
- 保持知識庫最新。過時的文件可能導致過時的回應。定期審閱和更新你的 project 知識。
- 撰寫清晰的指令。對你想要的事情要具體。模糊的指令導致不一致的結果。
- 為你的文件取描述性名稱（例如：「Q4-2025-Sales-Report.pdf」而不是「report.pdf」），並將相關檔案分組。Claude 使用檔案名稱和位置接近性來理解文件之間的關係。
- 按名稱參考文件。提問時，你可以提及具體的文件來幫助 Claude 聚焦搜尋：「根據我們的 Q3 報告，客戶最關心的問題是什麼？」

### 接下來

在下一課中，我們將學習如何使用 Artifacts 建立小型應用程式——Claude 建立並可以立即分享的實際輸出成果。

如需有關開始使用 projects 的更多資訊，請訪問 Anthropic Help Center。

## 使用 Artifacts 創作

### 學習目標

完成本課後，你將能夠：

- 說明什麼是 artifacts，以及 Claude 何時建立它們
- 與同事分享 artifacts 並公開發布
- 對常見的 artifact 問題進行疑難排解

Artifacts 是 Claude 在你對話旁邊的專用視窗中建立的獨立互動式輸出。你不會得到埋在聊天中的一大塊程式碼或文字，而是看到你的內容渲染後、可以立即使用的樣子——無論是一個可運行的網站、一個互動式圖表，還是一個你可以立即下載的文件。

當內容符合特定條件時，Claude 會自動建立 artifact：

- 它是重要且自成一體的，通常超過 15 行
- 這是你可能想要編輯、迭代或重複使用的內容
- 它代表了複雜的內容，無需周圍的對話就能獨立存在
- 這是你之後想要參考或使用的內容

### 常見的 Artifact 類型

Claude 可以建立不同類型的 artifacts，各自適合不同需求：

| Artifact 類型 | 說明 |
|--------------|------|
| 文件（markdown、純文字、Word 文件、PDF、PowerPoint、Excel） | 適合任何你想要匯出或繼續編輯的文字密集內容——如會議記錄、報告、專案計畫、部落格文章等。 |
| 程式碼片段 | 任何程式語言（Python、JavaScript、C++ 等）的可運行程式碼。你可以查看程式碼、複製它或下載到你自己的專案中使用。 |
| HTML 頁面 | 在單一檔案中包含 HTML、CSS 和 JavaScript 的完整網頁。非常適合登陸頁面、表單、互動式示範或快速原型。 |
| SVG 圖像 | 用於標誌、圖示、插圖和其他視覺元素的可縮放向量圖形。這些在 artifact 視窗中直接渲染，讓你精確看到你得到的是什麼。 |
| Mermaid 圖表 | 流程圖、序列圖、甘特圖、組織圖等。描述你想要可視化的關係，Claude 將建立一個你可以精煉的圖表。 |
| React 元件 | 具有真實功能的互動式 UI 元素——計算機、儀表板、遊戲、資料視覺化。這些不只是模型；它們包含實際邏輯並響應用戶輸入。 |

### 建立你的第一個 Artifact

建立 artifact 就像進行對話一樣簡單。只需描述你想要的內容，Claude 就會判斷是否以 artifact 呈現。

例如，你可能會說：

- 「建立一個顯示我們客戶入職流程的流程圖」
- 「建立一個互動式儀表板，讓我輸入每月支出並查看明細」
- 「為一個生產力應用程式設計一個帶有主要區塊和功能列表的登陸頁面」
- 「撰寫一個我可以為新計畫重複使用的專案簡報範本」

如果 Claude 在你期望的情況下沒有自動建立 artifact，你可以明確要求：「將此建立為 artifact」或「在 artifact 中給我看這個。」

當 Claude 生成一個 artifact 時，它會出現在你對話右側的專用視窗中。在這裡，你可以：

- 查看不同格式：在預覽（外觀）和底層程式碼之間切換
- 複製內容：點擊複製圖示，抓取內容以在其他地方使用
- 下載檔案：將 artifact 儲存為檔案到你的電腦
- 查看程式碼：精確看到 Claude 在底層生成了什麼

### 分享和發布 Artifacts

一旦你建立了有用的東西，你有幾個分享選項。

- 複製或下載：供個人使用或透過其他管道分享，使用 artifact 視窗右下角的複製或下載按鈕。
- 在你的組織內分享（Claude for Work）：Team 和 Enterprise 用戶可以在內部與同事分享 artifacts。共享的 artifact 保留在你的組織內，需要團隊驗證才能存取。
- 公開發布：對於 Free、Pro 和 Max 用戶，你可以發布 artifacts，讓任何有連結的人都能存取。當你發布時：
  - 只有選定的版本成為公開的（你的聊天保持私密）
  - 任何人都可以查看並與 artifact 互動，無需 Claude 帳號
  - 其他人可以「重混」你的 artifact——在自己的 Claude 對話中開啟它以修改和在其基礎上建立

若要發布，點擊 artifact 右上角的「Share」或「Publish」按鈕。你隨時可以透過返回那個 artifact 並移除公開存取來取消發布。

> 注意：當你發布一個 artifact 時，任何有連結的人都可以公開存取它——即使沒有 Claude 帳號也可以查看。已發布的 artifacts 不會被搜尋引擎索引，因此不會出現在 Google 結果中。

### 充分利用 Artifacts 的技巧

- 對你想要的內容要具體。「建立一個預算追蹤器」很好，但「建立一個每月預算追蹤器，讓我可以按類別輸入支出、查看圓餅圖明細，並在超出預算時獲得警告」更好。
- 描述最終用戶。告訴 Claude 誰會使用這個 artifact 有助於它做出適當的設計選擇。「這個流程圖是給新員工的」與「這個流程圖是給工程團隊的」會導致不同的結果。
- 逐步迭代。要求 Claude 一次添加一個功能或做一個改動。這讓識別有效的內容和及早發現問題變得更容易。
- 需要時請求 artifacts。如果你請求了一些實質性的內容，而 Claude 在聊天中回應而不是建立 artifact，只需說「請將此建立為 artifact」。

## 使用 Skills

### 學習目標

完成本課後，你將能夠：

- 說明什麼是 Skills，以及 Claude 如何使用它們
- 識別 Anthropic 用於文件建立的內建 Skills
- 在設定中啟用和管理 Skills

> 方案可用性：Skills 目前是 Pro、Max、Team 和 Enterprise 方案的功能預覽。如果你使用 Free 方案，可以閱讀了解概念，跳過實際操作步驟。

### 什麼是 Skills？

Skills 是 Claude 動態載入以提升在專業任務上表現的指令、腳本和資源資料夾。將它們視為專業知識套件——它們教導 Claude 如何以可重複的方式完成特定任務。

如果你曾使用 Claude 建立 Excel 試算表、PowerPoint 簡報、Word 文件或 PDF，你就已經看到 Skills 在運作了。那些檔案建立能力是由在幕後執行的 Skills 驅動的。但 Skills 遠不止於文件建立。自訂 Skills 可以將整個可重複的工作流程編碼化——季度差異分析方法、品牌語音審查流程或合規清單——讓 Claude 每次都遵循相同的嚴格步驟。

### Skills 的類型

你會遇到兩類 Skills：

- Anthropic Skills 由 Anthropic 建立和維護。這些包括針對 Excel、Word、PowerPoint 和 PDF 檔案的增強文件建立功能。Anthropic Skills 對所有付費用戶可用，Claude 在相關時自動調用它們——你不需要做任何特殊操作就能使用它們。
- 自訂 Skills 是你或你的組織為專業工作流程和領域特定任務建立的。例如，你可能建立一個將公司品牌規範應用於簡報的 skill、以特定格式構建會議記錄，或執行組織的資料分析工作流程。

### 啟用 Skills

Skills 目前對 Pro、Max、Team 和 Enterprise 方案的用戶作為功能預覽提供。要使用 Skills，你需要啟用程式碼執行和檔案建立，因為 Skills 需要 Claude 的安全沙盒計算環境才能運作。

啟用 Skills 的方式：

1. 前往「Settings > Capabilities」
2. 確保「Code execution and file creation」已切換開啟
3. 滾動至 Skills 區段
4. 根據需要切換個別 skills 的開關

對於 Enterprise 方案，組織所有者必須先在管理設定中啟用程式碼執行和 Skills，個別成員才能存取它們。

對於 Team 方案，此功能預覽在組織層級預設啟用。

啟用後，你將在設定中看到可用的 Skills 清單，包括 Anthropic 的內建 Skills 和你上傳的任何自訂 Skills。

### 在實踐中使用 Skills

Skills 的美妙之處在於你通常不需要考慮它們——Claude 根據你的請求自動處理 skill 選擇。以下是一些會調用 Skills 的提示範例：

- 「建立一個帶有總計公式的每月費用追蹤 Excel 試算表」
- 「將這份會議記錄文件轉成 PowerPoint 簡報」
- 「生成一份摘要這份資料的 PDF 報告」
- 「在 Excel 中建立一個帶有情境分析的財務模型」

當 Claude 使用 Skill 時，你會在 Claude 工作時的推理鏈中看到它被提及。輸出將是一個可以儲存到你電腦或直接儲存到 Google Drive 的可下載檔案。

### 檔案執行

同樣的能力意味著 Claude 可以使用你的實際檔案（在受限環境中）來建立你的檔案的更新版本。上傳幻燈片、試算表、合約（或任何 .xlsx、.pptx、.docx 或 .pdf 檔案），看著 Claude 建立幻燈片、進行分析並添加建議的編輯。Claude 完成後，你可以下載這些檔案或在 Drive 中開啟它們。

> 注意：在 Chat 中，Claude 建立文件的新版本，而不是就地編輯原始文件。要使用這些功能，你需要給 Claude 存取外部資料來源的權限。在提示時切換「Allow limited network access」即可。

### 安全考量

由於 Skills 可以包含可執行程式碼，請謹慎使用它們：

- 只從受信任的來源安裝自訂 Skills
- Anthropic 的內建 Skills 由 Anthropic 測試和維護
- 你上傳的自訂 Skills 對你的個人帳號是私有的
- 如果你從外部來源安裝自訂 Skill，在使用前審閱其內容以了解它做什麼

### 建立自訂 Skills

雖然 Anthropic 的內建 Skills 涵蓋了常見的文件建立任務，但 Skills 的真正力量來自於建立你自己的。自訂 Skills 讓你教導 Claude 你的特定工作流程、品牌規範和工作方式——這樣 Claude 就能在相關時自動應用這些知識。

建立自訂 Skill 的最簡單方式是透過與 Claude 本身的對話。你不需要撰寫程式碼或手動建立檔案——Claude 為你處理技術結構。

以下是如何透過對話建立 Skill：

1. 開始新的聊天並告訴 Claude 你想建立什麼。例如：「我想為撰寫季度業務審查建立一個 skill」或「我需要一個將我們的品牌規範應用於簡報的 skill。」
2. 回答 Claude 的問題。Claude 將訪談你的工作流程，詢問如：這個 skill 應該做什麼？對這類工作來說，好的輸出是什麼樣子的？你能舉例說明你何時會使用這個 skill 嗎？
3. 上傳參考資料（如果有的話）。範本、風格指南、品牌資產或你引以為傲的工作範例，都有助於 Claude 精確理解你在尋找什麼。
4. 儲存你的 skill。完成後，Claude 生成一個包含你正確結構化的 skill 的檔案。你只需儲存它，skill 就可以供 Claude 使用了。
5. 查看你的 skills。在左側側欄找到「Customize」標籤。在那裡你可以看到所有對你可用的 skills，甚至可以手動或透過與 Claude 聊天來編輯你使用的 skills。

你的自訂 Skill 將與 Anthropic 的內建 Skills 一起出現在你的 Skills 清單中。從那時起，每當你在相關任務上工作時，Claude 就會自動調用它——不需要手動觸發。你可以透過迭代改進你的 skills——請 Claude 編輯一個 skill，它將為你更新檔案。

### Skills vs. Projects

你可能會想——如果 skills 和 projects 都可以用來給 Claude 更多背景，我應該何時使用哪個？這樣想：projects 儲存知識，skills 執行任務。

| | Projects | Skills |
|--|---------|--------|
| 目的 | 儲存 Claude 參考的知識 | 定義 Claude 執行的流程 |
| 最適合 | 長期背景、參考資料、團隊協作 | 可重複的工作流程、多步驟任務、一致的方法論 |
| 範例 | 客戶中心、研究夥伴、回饋生成器 | 流程指引（如品牌或法律）、部落格起草、PDF 建立 |
| 持久性 | 知識在 project 的所有聊天中可用 | 指令在 skill 被調用時應用 |

這兩個功能互補。一個 skill 可以參考儲存在 project 中的知識——你的「客戶電話準備」skill 可能從上傳到 project 知識庫的客戶資料中提取。Project 提供「什麼」（資訊），skill 提供「如何」（流程）。

如需有關 Skills 的更多資訊，包括如何建立你自己的自訂 Skills，請訪問 Anthropic Help Center。

## 連接你的工具

### 學習目標

完成本課後，你將能夠：

- 說明什麼是 connectors，以及它們對你使用 Claude 的工作為何重要
- 導航 connectors 目錄並設定你的第一個連接
- 在與 Claude 的對話中有效使用連接的工具

### 什麼是 Connectors？

### 尋找和連接工具

Anthropic 在 claude.ai/directory 維護一個推薦 connectors 的目錄。目錄組織為兩個標籤：

- 網路：雲端服務和應用程式（Gmail、Notion、Slack、Asana、Linear、Stripe 等）
- 桌面擴充：透過 Claude 桌面應用程式在你電腦上執行的本地工具

若要瀏覽可用的 connectors，你也可以點擊聊天視窗左下角的「+」按鈕，然後選擇「Connectors」。

### 設定網路 Connector

以下是連接雲端服務的方式：

1. 找到 connector：導航至 claude.ai/directory，或在任何聊天中點擊「+ > Connectors」
2. 點擊 Connect：選擇你想新增的 connector
3. 驗證身份：你將被重定向至服務的登入頁面。用你現有的憑證登入
4. 授予權限：審閱 Claude 要求的具體權限，然後授權存取
5. 測試連接：返回 Claude 並嘗試一個簡單的請求，如「你能存取我的 [工具名稱] 嗎？」

一旦連接，根據你授予的權限，Claude 可以在該服務中搜尋、讀取，有時還可以執行操作。

### 桌面擴充

桌面擴充需要 Claude 桌面應用程式而非網頁介面。這些擴充讓 Claude 能與本地應用程式、你的檔案系統和 macOS 或 Windows 上的原生功能互動。

一些桌面擴充包括：

- 本地檔案存取，用於讀取和整理文件
- 瀏覽器控制，用於自動化網路任務
- 原生應用程式整合（如設計工作的 Figma）

安裝桌面擴充的方式：

1. 下載並安裝 Claude 桌面應用程式
2. 開啟應用程式並導航至「Settings > Extensions」
3. 瀏覽可用的擴充並點擊「Install」
4. 按照該擴充特定的額外設定步驟進行操作

### 在你的工作中使用 Connectors

連接你的工具後，Claude 在回應你的請求時會考慮它們。以下是一些使用連接工具的實際方式：

#### 專案管理（Asana、Linear、Jira）

- 「本週到期的最高優先級任務是什麼？」
- 「為審閱 Q4 預算提案建立一個新任務」
- 「摘要我們產品發布專案的狀態」

#### 溝通（Slack、Gmail）

- 「找到我們討論供應商合約的電子郵件串」
- 「起草一份對 #marketing 頻道最新訊息的回覆」
- 「昨天的討論中，團隊對時間表做了什麼決定？」

#### 文件（Notion、Google Drive、Confluence）

- 「搜尋我們文件中的品牌語音指南」
- 「摘要上週產品審查的會議記錄」
- 「我們的風格指南對使用縮寫詞有什麼說法？」

#### 業務工具（Stripe、PayPal、Salesforce）

- 「顯示過去一季的營收趨勢」
- 「Acme Corp 商機的狀態是什麼？」
- 「列出最近超過 1,000 美元的交易」

### 安全性和權限

當你將 Claude 連接至外部服務時，你授予它存取這些服務中的資料——有時還可以修改資料。以下是一些重要考量：

- 範圍存取：權限針對 connector 所需的內容，你可以在每個應用程式選單中開關個別權限。
- Claude 看到的是你能看到的：Claude 只能存取你有權存取的資料。連接你的工作電子郵件不會讓 Claude 存取你 CEO 的收件匣——只有你自己的。
- 可隨時撤銷：你可以透過 Claude 的設定或第三方服務的安全設定斷開服務連接。就像 Skills 一樣，你也可以找到或建立自訂 connectors。保持同樣的謹慎——只安裝來自受信任來源的 connectors。

## Enterprise Search

### 學習目標

完成本課後，你將能夠：

- 說明什麼是 Enterprise Search，以及 Enterprise Search 可以回答哪些類型的問題
- 了解管理員和用戶的設定流程
- 認識安全性和權限如何保護組織資料

> 方案可用性：Enterprise Search 適用於 Team 和 Enterprise 方案，必須由工作區管理員啟用。如果你使用 Free、Pro 或 Max 方案，可以跳過本課。

### 什麼是 Enterprise Search？

Enterprise Search 在你的側欄新增一個專用的「Ask {Your Org Name}」選項。它專門設計用於尋找和整合分散在你公司工具和資料來源中的知識。將 Enterprise Search 視為你整個組織的預建 Project——你公司的知識庫已經載入，因此你可以直接開始提問，獲得有背景感的回應。

與啟用了 connectors 的常規聊天不同，Enterprise Search 是專門為資訊蒐集設計的，使用由 Anthropic 團隊配置的自訂指令。

### 你可以問什麼？

Enterprise Search 對於跨多個來源的問題或需要整合整個組織資訊的問題特別有價值。以下是一些常見的使用案例：

#### 快速掌握最新情況

- 「我昨天不在，發生了什麼？」
- 「摘要過去一週業務中的重要更新」
- 「Platform 專案目前有哪些障礙？」

#### 政策和流程問題

- 「我們公司的遠端工作政策是什麼？」

### 設定 Enterprise Search

Enterprise Search 需要兩步驟設定流程：首先管理員為組織配置，然後個別用戶用自己的個人帳號驗證身份。

#### 管理員（所有者）

Enterprise Search project 對所有 Team 和 Enterprise 組織預設啟用，但所有者在團隊成員能夠使用之前需要完成初始設定：

1. 點擊左側側欄中的「Ask Your Org」。
2. 點擊「Set up for your org」繼續（或「Disable」關閉功能）。
3. 連接你組織的工具。你需要為「Documents」（如 Google Drive 或 SharePoint）和「Chat」（如 Slack 或 Microsoft Teams）選擇一個 connector。電子郵件建議但為可選。
4. 點擊「+ Add more」設定你的團隊所需的任何其他工具。
5. 自訂 project 名稱。你輸入的任何內容將在每個人的側欄中顯示為「Ask [Name]」。
6. 加入描述，然後點擊「Finish set up」。

設定完成後，project 對你組織的所有成員可用。

#### 用戶

1. 點擊側欄中的 project。
2. 按照引導式入職流程連接到推薦的服務。
3. 向你想要搜尋的每個服務驗證身份（Slack、Google、Microsoft 365 等）。
4. 開始向 Claude 提問關於你組織知識的問題。

你啟用的 connectors 越多，搜尋結果就越全面。你隨時可以透過點擊 project 指令區段中的「Connect」來新增更多 connectors。

## Research 模式的深度調查

### 學習目標

完成本課後，你將能夠：

- 說明 Research 的功能：系統性的多來源調查
- 識別何時使用 Research 進行全面的資訊蒐集
- 了解 Research 如何與延伸思考（extended thinking）搭配，以提供徹底的報告
- 為複雜調查撰寫有效的 Research 提示

### 什麼是 Research？

Research 是一個進階功能，將 Claude 從對話型助手轉變為系統性調查者。當你啟用 Research 時，Claude 不只是回答你的問題——它從多個角度探索問題，整合來自網路和你連接的整合服務的資訊。

將它想像成擁有一個能花幾個小時蒐集資訊、交叉參考來源並整理全面報告的熟練研究助理——只不過它在幾分鐘而不是幾個小時內完成。

Research 在你需要的不只是快速答案時特別有價值。它設計用於全面理解需要從多個來源整合資訊、比較不同觀點並將發現整合為可操作洞見的情況。

### 何時使用 Research

了解何時使用 Research 而不是其他 Claude 功能，有助於你獲得特定需求的最佳結果。

適合使用 Research 的情況：

- 需要整合多個來源資訊的全面報告
- 跨網路和你的連接整合服務（如 Google Workspace）的深度分析
- 通常需要數小時手動工作的徹底調查
- 比較分析，如評估競爭對手或供應商選項
- 帶有可驗證引用的報告

Research 理想適用於以下任務：

- 市場分析和競爭研究
- 規劃複雜專案，如團隊外出活動或產品發布
- 整合來自你的電子郵件、行事曆和文件的資訊
- 建立從多個來源汲取的技術文件
- 準備需要最新、經過驗證資訊的簡報

改用網路搜尋的情況：

- 你需要一個快速、具體的事實（如今天的股價或公司地址）
- 答案只需要一兩個來源
- 速度比全面性更重要

改用延伸思考的情況：

- 你需要對不需要外部資訊的複雜問題進行深度推理
- 你在處理數學問題、程式碼除錯或邏輯分析
- 答案來自於對問題的推理，而非蒐集資訊

改用 Enterprise Search 的情況：

- 你需要從你組織的內部知識汲取答案——文件、Slack 串、電子郵件、會議記錄
- 你正在入職，想快速找到你公司如何處理某件事（如政策、流程或過去的決策）
- 你在問一個針對你公司而非公開網路的問題

### Research 的運作方式

當你啟用 Research 時，你啟動了一個遠超過簡單網路搜尋的代理型、多步驟流程。Claude 根據已經找到的內容自主決定下一步搜尋什麼，追蹤線索並填補空白，無需你指引每一步。

1. Claude 規劃其方法。當 Research 啟用時，延伸思考自動啟動。這讓 Claude 能分解你的請求、識別所需資訊，並規劃如何調查你問題的不同角度。
2. Claude 進行多次搜尋。Claude 不是執行單一搜尋，而是進行相互建立的多次搜尋。它根據找到的內容決定下一步調查什麼，追蹤有前途的線索並填補空白。
3. Claude 整合發現。從多個來源蒐集資訊後——包括網路和任何連接的整合服務，如 Gmail、Google Calendar 或 Google Drive——Claude 將所有內容整合為全面、組織良好的報告。
4. Claude 提供引用。Research 報告中的每個主張都連結回其來源，讓驗證資訊和在需要時深入研究變得容易。

### 在實踐中使用 Research

以下是啟用和使用 Research 的方式：

1. 點擊聊天介面左下方的「+」按鈕
2. 從選單中選擇「Research」——一旦啟動，它會高亮顯示
3. 輸入你的提示並提交
4. Claude 將在後台工作，你會看到它搜尋和分析時的進度指示器

> 重要：Research 需要啟用網路搜尋才能運作。如果你還沒有開啟網路搜尋，可以從同一個「+」選單中開啟。

### 有效 Research 提示的技巧

由於 Research 根據複雜度可能需要 5 到 45 分鐘，投入時間精心製作你的提示是值得的。以下是一些策略：

- 對你的目標要具體。不要只說「告訴我電動車市場」，試試「分析電動車電池市場——識別關鍵參與者、技術趨勢和可能影響投資決策的供應鏈挑戰。」
- 指定你想要的區段或結構。Claude 將圍繞你提供的結構整理其發現。例如：「比較團隊外出活動的場地選項，包括：位置和交通便利性、會議空間和設施、餐飲選項和定價考量。」
- 包含相關限制。預算範圍、時間線、地理要求和其他參數幫助 Claude 將研究聚焦在相關選項上。
- 請 Claude 幫助完善你的提示。如果你不確定如何框架你的研究問題，你甚至可以在啟用功能之前請 Claude 幫你寫一個更好的 Research 提示。

### 使用連接的整合服務

當你連接了 Google Workspace 或其他整合服務時，Research 變得更加強大。Claude 可以從你的電子郵件、行事曆和文件中提取背景，並進行網路研究。

例如，你可能請 Claude：

- 「摘要在我的電子郵件和 Slack 中討論過的 X 專案的內容，然後研究類似計畫的產業最佳實踐」
- 「查看我下週的行事曆承諾，並研究我要會面的每家公司」
- 「找到所有關於我們定價策略的內部文件，並與競爭對手如何定位自己進行比較」

使用 Research 搭配整合服務時，你可以透過說「從我的 Google Drive 中提取相關背景」或「包含我最近關於這個主題的電子郵件洞見」來引導 Claude。

> 小技巧：你也可以關閉網路搜尋，只對你連接的工具進行內部研究——非常適合「我們的團隊在 Slack 和文件中討論了 Q3 發布的什麼內容？」之類的問題。

## 使用 Claude 的其他方式

### 學習目標

完成本課後，你將能夠：

- 了解何時使用其他 Claude 產品，包括 Claude Code、Claude for Slack、Claude for Excel、Claude for PowerPoint 和 Claude for Chrome

如我們在課程開始時提到的，Claude 是一個智能。Claude.ai 只是使用它的其中一種方式。

Claude 也在幾個專業工具中提供，這些工具設計為在你已經工作的地方與你相遇。本課介紹了另外五種使用 Claude 的方式，每種都針對特定工作流程和使用案例量身打造。

### Claude Code

Claude Code 是一個代理型程式碼工具，在你工作的地方工作——在你的終端機、IDE、瀏覽器，甚至在 Slack 中。它理解你的程式碼庫、執行命令，並通過自然語言處理整個開發工作流程。

適合使用 Claude Code 的情況：

- 你想通過描述你需要的內容來建立功能，讓 Claude 撰寫程式碼、執行測試並建立 commits
- 你需要通過貼上錯誤訊息來除錯問題，讓 Claude 分析你的程式碼庫以識別和修復問題
- 你在導航一個陌生的程式碼庫，想問關於不同部分如何協作的問題
- 你想自動化繁瑣的任務，如修復 lint 錯誤、解決合併衝突或撰寫發布說明
- 你偏好在終端機中工作，搭配你現有的 IDE 和開發工具，而不是切換到單獨的介面

### Claude in Slack

Claude 直接整合到 Slack，讓你可以在頻道和串流中獲得幫助，或將 Slack 背景帶入你的 Claude 對話。

適合使用 Claude in Slack 的情況：

- 你想在不離開 Slack 的情況下起草回覆、摘要冗長的串流或分解複雜的討論
- 你需要通過讓 Claude 整合你工作區中的相關對話和共享文件來為會議做準備
- 你正在入職一個新團隊，想通過審閱頻道歷史來理解進行中的專案
- 你想直接從錯誤報告或功能討論中交接程式碼任務——只需標記 @Claude，它就可以使用周圍的背景啟動 Claude Code 工作階段
- 你需要在對話中快速獲得關於產業趨勢、技術概念或公司資訊的解答

### Claude for Excel

Claude for Excel 通過側欄將 Claude 直接帶入 Microsoft Excel，讓你可以通過對話分析、理解和修改試算表。

適合使用 Claude for Excel 的情況：

- 你在使用一個複雜的多頁簽工作簿，想了解特定公式或跨表計算流程如何運作
- 你需要在保留公式依賴關係和關聯的同時，在整個模型中更新假設或輸入
- 你在除錯試算表錯誤，如 #REF!、#VALUE! 或循環參考，並希望 Claude 追蹤它們的來源並建議修復方法
- 你想建立新的試算表或在保持適當公式結構的同時填充現有範本的資料
- 你需要快速建立樞紐表或圖表來視覺化你的資料

### Claude for PowerPoint

Claude for PowerPoint 將 Claude 以側欄形式帶入 Microsoft PowerPoint，讓你可以通過對話起草、編輯和重構簡報，同時保持你現有的範本和品牌樣式不變。

適合使用 Claude for PowerPoint 的情況：

- 你想將大綱、文件或一組筆記轉成第一稿幻燈片，而不是手動建立每一張幻燈片
- 你需要重寫或精煉幻燈片文案——縮短項目符號、添加演講者備注或調整語氣以適應特定受眾
- 你在重構現有的簡報，想幫助重新排序區段、拆分密集的幻燈片或合併重疊的幻燈片
- 你想在整個簡報中應用一致的格式——標題、項目符號樣式和佈局——而不需要手動修復每一張幻燈片
- 你想要對一張幻燈片的快速視覺建議，例如哪種佈局或圖表類型最適合你要表達的要點

### Claude for Chrome

Claude for Chrome 是一個瀏覽器擴充，在 Google Chrome 中將 Claude 作為側欄添加。它可以觀察你正在做的事情並直接在你的瀏覽器中執行操作。

適合使用 Claude for Chrome 的情況：

- 你想在瀏覽時摘要文章、研究論文或網頁
- 你需要幫助起草電子郵件回覆或管理你的收件匣
- 你在填寫重複性表單，想自動化流程
- 你想測試網站功能或在多步驟工作流程中導航，而不用手動點擊每個步驟
- 你需要一個在你在標籤和任務之間移動時保持背景的瀏覽輔助工具。這讓它非常適合從利基內部工具、CRM 或儀表板中提取背景。

> 重要提示：Claude for Chrome 目前處於研究預覽階段。Anthropic 建議將其用於受信任網站上的低風險任務。在採取高風險操作（如購買或分享個人資料）前，擴充會要求許可，且某些類別的網站（金融服務、成人內容）預設被封鎖。

### 摘要

每一個工具都將 Claude 的能力擴展到你工作的特定環境中：

| 工具 | 最適合 | 執行位置 |
|------|-------|---------|
| Claude.ai | 一般任務、研究、寫作、分析、檔案建立 | 網頁、桌面和行動應用程式 |
| Claude Code | 軟體開發、程式碼庫導航、git 工作流程 | 終端機/命令列、IDE 或瀏覽器 |
