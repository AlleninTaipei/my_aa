## 2. Cowork 是什麼

Cowork 是 Claude 桌面應用程式內的一種模式，Claude 在這個模式下替你執行任務。你指定一個電腦上的資料夾、連接工作所在的應用程式（Gmail、Slack、Google Drive、行事曆），然後描述你要完成什麼。Claude 制定計畫、逐步執行、使用所需的工具和檔案，最後將真實的可交付成果儲存回你的資料夾。

最重要的心智模式：Cowork 的核心是委派，而非單純聊天。Chat 在思考問題、起草與精修、腦力激盪、提問時依然非常有用，Cowork 並不取代這些。Cowork 新增的是：把一整件工作——蒐集背景資料、分析、產出檔案、工具操作——全部交給 Claude，讓它完成後再回來。

### 在你的環境中運作

Cowork 在你的環境中運作，而非僅僅旁邊提供建議：

- 你的檔案：Claude 讀取你指定資料夾內的檔案，將完成的輸出寫回同一位置
- 你的應用程式：Claude 從已連接的電子郵件、行事曆、通訊、雲端硬碟、CRM 中取得背景資料
- 你的瀏覽器：對於沒有連接器的網頁工具，Chrome 中的 Claude 可以直接讀取並操作頁面
- 你的工具：Claude 採取行動，而不只是描述該怎麼做

### Chat、Cowork、Code 的比較

| 模式 | 核心定位 | 最適用場景 |
|------|----------|------------|
| Chat | 與 Claude 一起思考 | 提問與探索想法、起草後自行打磨、任何能在單一回應中完成的事 |
| Cowork | 委派工作給 Claude | 跨檔案與工具的多步驟工作、產出真實可交付成果、想交出去再回頭查看的工作 |
| Code | 與 Claude 一起開發軟體 | 在程式碼庫中工作、跨多個源碼檔案的建置或重構、開發者工作而非文件工作 |

## 3. 設定 Cowork

### 安裝

Cowork 在 Mac 和 Windows 的 Claude 桌面應用程式中執行。從 claude.com/download 安裝，開啟應用程式並登入後，在頁面右上角的模式選擇器中找到 Cowork。若 Cowork 未出現，可能需要付費方案或更新版本。

### 指定工作資料夾

在提示列中點選「Work in a project」，選擇電腦上的一個資料夾。這是每個新任務中最重要的設定，因為資料夾就是工作所在的地方。Claude 讀取內部所有檔案（Word 文件、Excel 檔、PDF、PowerPoint 等），並將完成的輸出儲存回同一位置。

選擇範圍限縮在一個專案或工作流程的資料夾。Claude 不需要存取你的整個文件資料夾，只需要包含該任務所需檔案的資料夾。

資料夾是 Cowork 的讀寫範圍：Claude 可以開啟你的檔案、編輯它們、建立新檔案、整理它們。這是與 Chat 最大的差異：Chat 可以讀取你上傳的內容，但無法將任何東西儲存回你的電腦；在 Cowork 中，Claude 可以。

雲端檔案的行為有所不同：連接器允許 Claude 做什麼因服務而異。許多服務（如預設的 Google Drive 和 M365 連接器）只能讀取和搜尋，其他服務可以建立或編輯。建立和迭代文件最可靠的地方仍是本地工作資料夾。

### 加入連接器

連接器是 Claude 進入工作所在應用程式的管道。在「Customize」區域設定一次後，即可按需開關各個連接器。

大多數人優先設定的連接器：

- 電子郵件與行事曆（M365 的 Outlook 或 Gmail）：用於從會議中提取背景資料、起草後續追蹤、尋找過去的郵件串
- 通訊（M365 的 Slack 或 Teams）：用於搜尋頻道歷史記錄、整理團隊討論內容
- 雲端儲存（Microsoft 365 的 SharePoint 或 OneDrive、Google Drive、Box）：用於存取不在本地電腦上的文件
- CRM 與專案工具：Notion、Salesforce、HubSpot、Asana、Linear 等，視團隊使用情況而定

連接器啟用後，在提示詞中自然地提及即可。「看看團隊在 Slack 上對這次發布說了什麼」或「找上季的客戶跟進郵件」，Claude 知道去哪裡找。

每個任務只會使用你已開啟的連接器。

對於沒有連接器的工具（內部儀表板、供應商入口網站、需要登入的網頁工具），Chrome 中的 Claude 是對應的解決方案，詳見第 9 課。

### 權限模式

指定資料夾後，Claude 自動取得讀寫授權。在資料夾內的任務執行方面，Cowork 有兩種權限模式：

| 模式 | 行為 | 適用情境 |
|------|------|----------|
| Ask before acting（預設） | 每次即將觸及外部的動作（發送電子郵件、發布訊息、分享檔案）前暫停等待確認 | 新工具、不熟悉的檔案、或任何想密切監看的任務 |
| Act without asking | 不暫停等待確認，速度更快但風險更高 | 主動監看時，且使用信任的檔案與網站 |

無論哪種模式，Cowork 永遠會在永久刪除檔案前詢問，這個提示無法跳過。

### 實作練習

1. 指定一個有真實工作內容的資料夾（選擇現有專案或本週的工作資料夾）
2. 連接一個應用程式（從電子郵件或雲端儲存開始）
3. 請 Claude 整理資料夾內容：「看看這個資料夾裡的所有內容，為我寫一份簡報，說明你學到了什麼——資料夾裡有什麼、文件之間的關聯、任何值得注意的洞察或其他你認為有參考價值的資訊。」
4. 對照自己的認知閱讀摘要：Claude 發現了什麼你可能遺漏的？它遺漏或理解錯誤了什麼？

## 4. Cowork 的三種適用工作模式

並非所有任務都適合 Cowork。以下三種模式涵蓋了 Cowork 最擅長處理的大多數工作。

### 模式一：多步驟任務

如果一項任務需要多個步驟，就適合 Cowork。用一個提示詞，Claude 就能蒐集背景資料、比較來源、進行額外研究、起草摘要，並格式化可交付成果。

範例：
- 將一週的客戶回饋郵件整理成主題，並附上各主題的引用範例
- 從三份報告和一份試算表中提取數字，再組合成一個互動式儀表板

### 模式二：工作在真實檔案中

當任務的目標是輸出一個真實成果物——Word 文件、試算表、簡報、PDF——並以所需格式儲存時，Cowork 非常適合。輸入也是真實的：Claude 讀取資料夾中既有的檔案、編輯它們，並將更改儲存回去。Chat 可以生成一個新文件；Cowork 則是在你已有的檔案上工作。

範例：
- 從標準模板和會議記錄中整理出格式化的客戶提案
- 從原始試算表組合出每月指標報告，並附上圖表

### 模式三：跨工具任務

工作涉及你已在使用的系統——Gmail、Slack、M365 套件、行事曆、CRM。Claude 通過連接器存取這些系統，在單次委派中規劃並執行整個流程，而不需要你一個提示詞一個提示詞地拼接步驟。

範例：
- 根據行事曆邀請、與會者名單和會議記錄，在 Outlook 起草跟進郵件
- 搜尋 Slack 中關於某次發布的所有團隊討論，整合成一份更新報告

### 排程任務

排程任務讓你設定一次，由 Claude 按週期執行。

設定方式有兩種：
- 從頭設定：輸入 `/schedule`，描述任務和週期，Claude 為你起草提示詞供你審閱確認
- 更自然的方式：先在 Cowork 中執行一次任務、確認輸出正確，再輸入 `/schedule` 將這個流程轉為週期性任務，指定週期（每小時、每天、工作日或手動），審閱後確認

排程任務只在電腦開機且 Claude 執行時運作。若筆電蓋上、睡眠或關機，Claude 會在你回來後補執行，並告知你有延遲。

適合排程的例子：
- 週五下午的回顧：本週完成的工作、Slack 或 Teams 主要決策摘要、下週任務預覽
- 每月一日從試算表提取的每月指標彙整
- 利用資料夾中的過去會議記錄和電子郵件串，為行事曆上的會議準備背景資料的每日早報

### Dispatch：從手機發送任務

Dispatch 讓你從 Claude 手機應用程式發起 Cowork 任務。你在手機上指派任務，工作在桌面上執行——使用相同的檔案、連接器和權限；完成後收到推播通知。

與排程任務一樣，Dispatch 只在桌機開機、清醒且已開啟 Cowork 的 Claude 時運作。

Dispatch 目前處於研究預覽階段，適用於 Pro 和 Max 方案。Enterprise 方案可能尚未開放，且由 IT 管理員設定。

## 5. 交付第一個任務

### 撰寫好的提示詞

好的 Cowork 提示詞做到三件事：

- 命名可交付成果：「一頁簡報」、「QBR 的投影片」、「附備注的候選人排序清單」。對格式和長度的具體說明能省去重新生成的需要
- 命名輸入來源：哪個資料夾、哪些頻道、哪個日期範圍、哪個應用程式。Cowork 的表現取決於你給它的背景資料
- 命名額外細節或 Claude 完成工作所需的細節：「我想看到基準、最好和最差情境，也要考量我們上季度新開的 3 個地點。」這句話告訴 Claude 你要的輸出類型，並提供 Claude 可能看不到的專業洞察

如果你習慣向聊天機器人提示，這感覺需要更多前期工作——確實如此。換來的是，大多數來回溝通發生在 Claude 開始工作之前，而不是橫跨五輪「其實，你能不能也...」

### 回答釐清問題

Claude 在正式開始工作前會提問，以釐清目標的模糊之處。在 Chat 中，你邊進行邊補充背景；在 Cowork 中，你委派工作，Claude 帶著完成的成果物回來。任何可能讓它困惑的地方——任何理解上的缺口——都會在深入工作前通過問題浮現。

大多數問題很簡單：Claude 提供幾個選項，你點選符合的那個。若沒有合適的選項，你可以用自己的話回答。

問題的作用是彌補理解缺口，例如：
- 格式缺口：「我看到資料夾裡有兩份備忘錄——哪一份是我要參考的格式？」
- 範圍缺口：「地區細分需要涵蓋多少內容？」
- 受眾缺口：「領導者會如何閱讀這份備忘錄？」
- 嚴謹性缺口：「當內容無法從來源 PDF 驗證時，該如何處理？」

### 中途調整方向

在 Claude 工作時監看計畫和進度。如果跑偏了——錯誤的來源、錯誤的格式、錯誤的語氣——請介入。你也可以在大幅跑偏時停止任務，精修提示詞後重新開始。

Chat 中大多數人的直覺是等回應完成後再重新生成——在 Cowork 中請抵制這種做法。Cowork 為中途修正而設計，重新定向的成本很低。

### 審查完成的成果物

當 Claude 完成後，你可以在應用程式中預覽檔案，檔案也同時存在於你的資料夾中。

以審閱你信任但還不完全了解的同事作品的方式審閱成果——需要開啟你的辨別力：

- 是否符合實際目標？還是產出了與你所求微妙不同的東西？
- 事實是否準確？若 Claude 引用了事實或數字，請對照來源材料確認。從請 Claude 列出引用來源開始，然後自行查核
- 有沒有任何看起來是虛構的內容？無法追溯到輸入來源的具體日期、名稱或引用是警示訊號，不是特色

如果草稿大致正確，告訴 Claude 要改什麼，而不是從頭開始。Claude 記得這次對話，編輯比重新生成快。

如果草稿在核心方面出錯，代表提示詞缺少了關鍵的背景資料。將新的背景指向 Claude，請它進行調整。

## 6. 讓 Cowork 越用越好的四種機制

完成第一個任務只是開始。越多 Claude 了解你、你的工作和你的團隊方式，下一個可交付成果就越好，而且完成得越快。

四種機制讓這成為可能，每種都給 Claude 可以延續的不同類型知識：

| 機制 | Claude 學到什麼 | 解鎖什麼 |
|------|----------------|----------|
| 全域指令 | 你是誰、你怎麼工作 | 每個任務從一開始就已根據你的角色、格式偏好校準 |
| 專案 | 一項工作流程的背景資料 | 在專案內，Cowork 像已在團隊中的人一樣工作，能存取相關檔案、歷史記錄和到目前的決策 |
| 技能 | 特定流程應如何執行 | 當符合的任務出現時，Cowork 以你的團隊方式執行：你的模板、你的標準、你的步驟 |
| 外掛 | 你的角色或領域的專業知識 | Cowork 從通才變成專家，具備你職位所需的工作流程、連接器和專業知識 |

這四者相互獨立，但可以疊加。第一天不需要全部設定好。隨著你在 Cowork 中的使用增加，你會察覺哪些全域指令適合套用到每次對話、哪些週期性工作流程適合成為專案、哪些流程值得建立技能，進而形成適合分享的外掛。

> **What's next:** 接下來三課依序介紹這四層機制。第 6 課涵蓋全域指令與專案；第 7、8 課涵蓋技能與外掛。

## 7. 全域指令與專案

### 全域指令

全域指令是一份常備簡報。你在「設定」中寫一次，Claude 就會在每個 session 中參照——每次聊天、每個排程任務、每次 Dispatch。

設定路徑：設定 → Cowork → 點選「全域指令」旁的「編輯」→ 撰寫並儲存。

要寫什麼：
- 你是誰、你做什麼
- 你使用的縮寫和簡稱（讓 Claude 不必每次問「QBR 是什麼」）
- 你希望輸出的呈現方式（格式、長度、語氣）

全域指令不必在第一天就完整。隨著你更多使用 Cowork，你會注意到你一直在給的糾正（「將結論放在最前面」或「不要用 Oxford comma」）——這些就是全域指令的候選內容。

以角色為例的全域指令結構：

| 角色 | 全域指令重點 |
|------|------------|
| 產品經理 | 公司規模與產品、常見縮寫、輸出格式（PRD、Launch Brief）、引用來源的規則、標記假設的要求 |
| 財務 | 報告對象、重要性門檻（如方差大於 5% 請標記）、備忘錄格式、顯示計算過程的要求 |
| 法務 | 合約類型、紅線慣例、審閱重點、所有輸出需加「草稿，請於發送前審查」 |

### 專案

全域指令涵蓋你這個人，專案則涵蓋你正在做的工作。

專案是綁定到特定工作流程的工作區——一個客戶、一個週期性可交付成果、一次發布。一個專案內有三個你可以設定的項目，以及一個 Claude 自行建立的項目：

- 指令：類似全域指令，但限定在這個專案。（「這個專案用於每週轉型團隊會議。目的是從整個組織蒐集資料，每週提煉成一份投影片。」）
- 排程任務：你設定的週期性執行，綁定到專案：週五的狀態彙整、會前簡報、月底彙整。在專案內的對話中設定，每次執行都帶入專案背景
- 背景資料：一個或多個 Claude 應從中工作的資料夾或連結。專案內的每個對話都可以存取它們
- 記憶：Claude 從你在專案內的對話中學到的東西。隨著你工作而積累，不需要你撰寫

記憶是專案帶來的差異。在專案外，除了全域指令外每個 session 都重新開始。在專案內，每次對話都為 Claude 的知識加入更多——讓下一個任務開始時，客戶的情況、上週的決策和未完成的事項就已在手邊。

### 哪些工作適合建專案

- 客戶或帳戶：一個客戶的會議記錄和可交付成果資料夾，以及該帳戶的常備指令。每次的會議準備、後續跟進和審閱草稿都在專案內進行，Claude 隨時間建立對關係的理解
- 週期性可交付成果：每月報告、季度評審、每週領導更新。同一個專案內的每個新週期是一個新對話，自動建立在上次洞察之上
- 一次發布或計畫：重新設計、遷移、新專案推出。簡報、決策、狀態更新——整個工作線索在一個地方，直到交付

### 三種建立專案的方式

| 方式 | 適用情境 |
|------|----------|
| 從頭開始 | 先空白建立，邊做邊加入指令和背景資料 |
| 從電腦上的既有資料夾 | 指定一個現有工作資料夾，該資料夾成為專案的工作目錄 |
| 從 Chat 專案遷移 | 若已在 Chat 中使用專案，可將指令和知識遷移到 Cowork（單向，Cowork 的變更不會同步回 Chat） |

在 Cowork 側邊欄點選「Projects」，然後選擇「New project」並選擇最適合你的設定方式。工作資料夾、指令和連接器可以隨時更改。

### 實作練習

1. 撰寫第一版全域指令區塊（幾句話即可，先不必完整）

   參考起始模板：
   ```
   我是一位 [職位] 在 [公司]，主要負責 [1-2項工作流程]。
   以下是你需要知道的常用 [縮寫/簡稱]。我的大多數可交付成果是 [文件、簡報、摘要、模型——視情況]。
   我希望更新和摘要簡潔——先說結論，背景限一段。我喜歡在建立最終文件前先審閱 .md 預覽。
   ```

2. 為一項工作流程建立一個專案（選擇週期性的事項——一個客戶帳戶、每月可交付成果、目前的計畫）

## 8. 技能

### 什麼是技能

技能是一個可重用的策略手冊——一個檔案和資源的資料夾——教 Claude 如何按你希望的方式執行特定類型的工作。當你開始一個符合技能的任務時，Claude 會載入這個手冊並遵循它。

技能在需要時自動使用，不需要你明確呼叫。Claude 會注意任務是否符合你已安裝的技能並自動載入。你也可以明確指定（「使用董事會備忘錄起草技能」）。

### 技能的四種組成

| 組成 | 用途 |
|------|------|
| 指令（SKILL.md 檔案） | 告訴 Claude 這個技能的用途、何時使用、以及如何執行。像為新同事寫的操作手冊一樣撰寫——具體到他們能完成工作 |
| 資產 | 標誌、品牌模板、投影片母版、字型。技能用來產出真實外觀成果的原始材料 |
| 參考資料 | 好輸出的範例、風格指南、條款庫、你會交給新隊友作為參考標準的過去作品。這是 Claude 學習「好」的標準的方式 |
| 腳本 | 讓 Claude 執行的小段程式碼，用於流程中應每次以相同方式執行的部分——差異計算、結構化比較、圖表格式、文件重新格式化 |

技能可以使用這些組成的任意組合。有些技能只有一個 SKILL.md 檔案，對某些流程已經夠用；有些包含 SKILL.md 加上品牌資產資料夾；有些包含全部四種。組合跟著工作走：包含需要的內容，不多不少。

以三個技能為例說明組成差異：

| 技能 | 組成 | 用途 |
|------|------|------|
| meeting-recap | 只有 SKILL.md | 固定一次性格式規則——決策、行動項目（負責人+日期）、未解決問題，150 字以內 |
| board-memo | SKILL.md + 資產（品牌模板）+ 參考資料（過去的備忘錄） | 將不成文的規則編碼——章節順序、每次都會被問到的問題；範本讓 Cowork 能比對 |
| variance-analysis | SKILL.md + 腳本（variance.py） | 固定每次不想重新推導的計算；腳本負責數學，Cowork 負責撰寫報告 |

### 建立技能

最快的方式是與 Claude 一起建立。在 Cowork 中開啟新對話，說類似：

```
我想為 [你不想再重複解釋的週期性流程] 建立一個技能。告訴我你需要知道什麼。
```

Claude 會提幾個問題：技能應做什麼、何時觸發、好的輸出長什麼樣、應參考哪些資源。盡可能具體回答——指向真實的工作範例、真實的模板、真實的過往輸出。輸出是一個包含 SKILL.md 和所需資產、參考資料和腳本的技能資料夾，可以直接安裝。

安裝後在「Customize」中找到技能。若要更改，在對話中指出修改需求並要求更新。Claude 直接更新技能。

技能在任何對話中（包括專案內的對話）以相同方式運作。

> **What's next:** 技能將你的特定工作流程打包，讓團隊中的任何人都能執行並獲得相同品質的結果。外掛將多個技能和連接器打包成一個可安裝的套件，圍繞一項工作職能建立。這是下一課的內容。

## 9. 外掛

### 外掛是什麼

外掛是圍繞工作職能建立的一套技能打包。一個技能是一本策略手冊；一個外掛是多本——多個技能，加上它們依賴的連接器和子代理人（sub-agents：技能可以啟動的專門輔助工具，在自己的背景中處理工作的一部分——例如研究步驟的研究子代理人、起草步驟的起草子代理人）。

外掛教 Claude 你的團隊的工作方式。安裝一個財務外掛，Claude 就知道你的團隊分析股票的方式；安裝法務外掛，Claude 就知道你的合約策略手冊。專業知識跟著安裝走，而不是跟著人走。

Anthropic 為知識工作中最常見的角色發布外掛——財務、法務、業務、行銷、客戶支援、產品管理等。

### 兩種外掛形式

端對端流程包：
當工作有許多順序步驟時，可以將各步驟的技能打包成一個外掛，讓整個流程一體執行。例如每月結帳外掛可能包含提取實際數字、建立差異表和起草董事會備忘錄的各個技能——每一個都是更大工作流程中的一步。

常用技能包：
這適合一組週期性任務。你可以將最重要的任務打包成一個外掛。例如財務外掛可能包含差異分析、財務建模、投資備忘錄起草和季度報告的技能。它們彼此獨立，只是團隊最常用的技能組合。打包後，新隊友安裝一個就有了整個團隊的工具箱。

### 從 Anthropic 市集安裝外掛

在 Cowork 的「Customize → Plugins」中找到市集外掛。瀏覽符合你工作的外掛、點選「Install」並授權外掛使用的連接器。外掛的技能立即可用。

### 客製化外掛

市集外掛是強大的預設值，不是最終答案。技能和連接器使用通用版本的工作流程；你的團隊有自己的模板、定義和步驟。

安裝後，前往「Customize → Plugins → [外掛名稱]」點選「Customize」。這會開啟一個新的 Cowork 任務，與 Claude 一起調整外掛。你可以指向具體資產、分享背景資料或上傳你希望作為基礎的範例，Claude 會更新外掛以符合你的團隊實際情況。

例如：
```
這是我們最後三份紅線 NDA。更新這個外掛的 /nda-triage 技能，讓格式和語氣與這些相符。
```

### 自行建立外掛

若你的團隊有不符合現有外掛的工作流程，可以通過 Cowork 建立一個。Cowork 會打包工作流程所需的技能、加入它依賴的連接器，並將其包裝以便輕鬆安裝。

大多數團隊從小規模開始：一個最重複性任務的技能，然後再加一個。當有三四個技能和重要的連接器時，就是一個值得分享的外掛。

### 實作練習

在新的 Cowork 對話中輸入：

```
/setup-cowork
```

技能會開啟一個簡短訪談，Claude 詢問你的工作類型，然後建議最適合你需求的外掛。你可以直接從對話中安裝外掛並進行測試。安裝後，為你的團隊進行客製化。

## 10. Chrome 中的 Claude

Chrome 中的 Claude 是沒有連接器的工具的橋梁。對於任何存在於瀏覽器中的內容，Claude 都可以讀取並操作這些頁面。

Claude 可以與 Cowork 協同工作：在瀏覽器中蒐集資訊並採取行動，然後將結果帶回 Cowork 建立可交付成果。一次對話，兩個介面。預設情況下，Chrome 中的 Claude 在敏感動作前會詢問，你可以逐一核准或拒絕。

### 適用工作場景

- 內部儀表板：財務團隊的 Tableau 視圖、業務營運的 Looker 儀表板、你每週一查看的 BI 工具。Claude 可以提取數字、下載到你的電腦，並將這些背景資料用於 Cowork 任務
- 供應商入口網站與客戶系統：沒有 API 的採購入口網站、單一登入後的 CRM、需要逐一分類的客戶支援工具。Claude 可以瀏覽入口網站、提取所需內容，並按你的方式操作
- 需要登入的網頁應用程式：任何有瀏覽器介面的工具——即使是你的團隊自己建立的——都變得可程式化
- 網路研究並產出可交付成果：開啟十個分頁、提取各頁面的內容，整理成摘要——無需複製貼上

模式：任何時候你想「我希望能把這個背景資料給 Claude，但它在網頁上」，Chrome 中的 Claude 就是答案。

### 與 Cowork 搭配使用的範例

你的團隊的客戶健康儀表板需要登入、沒有連接器，你想在週五前獲得每個顯示黃色或紅色帳戶的一頁摘要。

在 Cowork 中說：

```
在 Chrome 中開啟客戶健康儀表板，提取所有顯示黃色或紅色的帳戶，
對於每個帳戶，從 Drive 的客戶資料夾中提取過去 30 天的活動記錄和 
#customer-success Slack 中的最近討論串。
建立一份我可以在週五電話前審閱的一頁摘要。
```

Cowork 將瀏覽器步驟交給 Chrome 中的 Claude，取回資料，從 Drive 和 Slack 提取支援背景，並在你的資料夾中建立摘要。一次委派，三個背景資料來源。

### 注意事項

- 你需要已登入：Claude 無法替你登入工具。若儀表板需要驗證，你在瀏覽器中登入一次，Claude 在你已驗證的 session 中工作
- 謹慎授權 Claude 在網頁上存取的範圍：Claude 看到你看到的內容。對於敏感網站，縮小 Claude 的操作範圍，並在核准動作前審閱

設定指引參見 claude.com 的支援文件「Get started with Claude in Chrome」。

## 11. Claude for Microsoft 365

Claude 以增益集形式存在於 Word、Excel、PowerPoint 和 Outlook 內部，直接在你開啟的檔案和空間中工作。一次對話可以在應用程式之間傳遞背景資料：在 Excel 中建立分析，然後交給 PowerPoint 製作投影片；用 Word 備忘錄作為 Outlook 回覆草稿的來源。

### 每個應用程式中 Claude 的功能

Excel：
- 分析資料、撰寫公式、除錯 #REF! 錯誤和循環引用
- 在不破壞模型的情況下執行情境測試
- 從模板建立試算表、引用特定儲存格
- 最強用法：「從 Q3 工作表提取實際數字，與同一個工作簿中的 Q3 計畫進行比較，並在每一行旁的 F 欄寫差異說明。」

PowerPoint：
- 讀取你的投影片母版、字型和色彩方案，然後建立與之相符的投影片
- 生成原生的可編輯圖表（而非貼上的圖片）
- 操作你選取的任何投影片
- 最強用法：「把我剛在 Excel 中做的分析，用我們的簡報模板做成 QBR 的三頁投影片。」

Word：
- 直接在文件中起草、修改和重新格式化
- 處理批注和追蹤修訂
- 從已連接的來源提取背景資料來支撐草稿
- 最強用法：「根據這份備忘錄的正文和附錄中參照的來源資料，起草執行摘要。」

Outlook：
- 以其他工作的背景資料分類收件匣
- 起草反映過去討論串、行事曆背景和近期決策的回覆

### 跨應用程式的工作流程範例

Outlook → Word → Excel → PowerPoint → 回到 Outlook：

1. 客戶簡報到達收件匣。「在 Word 中開啟簡報，用我們的公司模板開始備忘錄。」Word 開啟附件，並已在側邊欄載入郵件串
2. 寫備忘錄途中需要數字。「建立選項 2 背後的市場規模模型。」Excel 開啟，從 Claude 剛在 Word 中讀取的簡報提取假設，建立含有可查核公式的多頁模型
3. 「把這個做成客戶模板格式的指導委員會投影片。」簡報在你的投影片母版中建立，附上 Excel 中更新數字的原生可編輯圖表
4. 「在週四前找 30 分鐘和團隊開會。」邀請附上與會者待你點選傳送

### 何時用 M365 vs. Cowork

| 情況 | 適用工具 |
|------|----------|
| 工作需要從多個來源提取，最終輸出為可交付成果 | Cowork |
| 在 Office 檔案本身中工作——就地編輯，並將背景資料從一個應用程式帶到另一個 | Claude inside M365 |

大多數真實工作兩者都會用到：Claude 建立第一份投影片草稿，你在 PowerPoint 中開啟並用 Claude 精修；Claude 將資料放入模型，你在 Excel 中開啟並請 Claude 研究你發現的異常值。兩個介面無縫交接，檔案在兩者中都是同一個檔案。

設定指引分別參見 claude.com 支援文件的「Work across Excel, PowerPoint, and Word」和「Claude for Outlook」。

## 12. 安全使用 Cowork

### 設定層面的保護

最大槓桿的動作是你指定 Claude 的資料夾——這是 Claude 可以讀取、寫入和（經你確認後）刪除的範圍。

- 使用專用的工作資料夾，而非全包式的資料夾：把 Claude 指向 Documents、Downloads 或 Desktop，等於讓新同事翻遍你的所有檔案。建立一個工作資料夾，移入（或複製）所需的內容，將 Claude 指向那裡
- 在開始前備份任何無可取代的內容：如果一個檔案很重要、而且無法重新生成——舊客戶交付成果、無法重新取得的合約、任何遺失會後悔的東西——確保副本存放在 Cowork 無法觸及的地方：雲端備份、獨立資料夾、未連接的硬碟。Claude 在刪除前會詢問，但不慎點擊錯誤確認的代價就是這個檔案
- 先用副本測試新工作流程：例如建立每週五執行的排程任務，第一次先針對資料副本執行。確認行為正確後，再指向正式資料夾

### 提示詞層面的防護

- 對破壞性動詞要明確：「裁切這個段落」可以理解為「從畫面中移除」或「從檔案中刪除」；「更新這個檔案」可以意味著「重寫它」或「新增到它」。如果錯誤的理解會造成無法挽回的後果，說清楚你的意思：「從草稿中移除這個段落，但保留檔案。」「新增一個附錄；不要重寫現有段落。」
- 在提示詞中指定範圍：「只限這個資料夾中最近 3 個更新的檔案。」「只限 Q3 結案的合約。」「不要傳送給任何人——只起草。」這縮小了 Cowork 的操作範圍，也給你一個清楚的基準來判斷是否偏離
- 排程任務初期先輸出草稿：排程任務在你不監看時執行。在你確信任務按需要運作之前，提示它起草供你審閱，而不是代你直接發送

### 執行過程中的三個檢查

讀一遍計畫：當 Claude 開始任務時，它在進度標籤中列出它要做的事。快速瀏覽：計畫有意義嗎？步驟順序正確嗎？它使用的是正確的來源嗎？必要時重新引導。

留意意外模式：你不需要驗證每個命令。但如果 Claude 碰觸了你沒提到的檔案或網站，或範圍蔓延超出你的要求，停止任務。「感覺哪裡不對」是真實的訊號——留意它。

謹慎核准確認提示：對任何會發送、發布或分享的動作保持在「Ask before acting」模式——當確認提示出現時，請讀它。大多數錯誤不是因為安全措施失效；它們是因為有人點擊了不完全是他們意圖的動作的確認。這個對話框存在是因為這個動作很重要——請如此對待它。

### 不適用 Cowork 的情況

- 需要稽核記錄的受規範工作流程：Cowork 活動不會被捕捉在稽核日誌、合規 API 或資料匯出中
- 任何你不會信任一位聰明但不完全了解的同事在無監督下完成的事：把法律文件發送給對方、發布公開公告、推送面向客戶的變更。Claude 可以準備；你負責執行
- IT 團隊未明確批准的範圍之外的高度敏感個人資料

更多安全使用指引參見 claude.com 支援文件的「Use Claude Cowork safely」。

## 13. 驗證技能與外掛

### 為什麼需要驗證

當你建立技能或將技能打包成外掛，你實際上在建立一個小產品，讓其他人使用。就像你會交給同事的任何東西——模板、試算表模型、清單——值得在離開你的桌面前先試跑一遍。

你使用自己建立的技能時，知道如何繞過問題或失敗；你知道要問什麼、要給它什麼檔案，以及答案應該是什麼樣子。隊友沒有這些。他們可能措辭稍有不同、給它略有差異的輸入，或遇到邊緣案例——一個真實但略超出技能設計範圍的情況。技能容易在這些地方卡住，而使用者不會知道原因。

用評估測試技能（eval，即 evaluation 的縮寫），就是在別人遇到這些問題之前先自己發現。不要被這個詞嚇到。eval 只是一次試跑：一個真實的請求進去，你看看輸出，告訴 Claude 要修正什麼。無需程式碼，無需測試腳本——只需要你對結果是否足夠好的判斷。

### 評估系統的運作方式

當你用 skill-creator（Claude 的內建技能建立助手）建立技能時，它會將 eval 作為流程的一部分。

Skill-creator 提出兩個或更多有人會用你的技能的真實提示詞。對於每個提示詞，它產出一對輸出：

- 一個 Claude 使用你的技能時的輸出
- 一個 Claude 用相同提示詞但不使用你的技能時的輸出

第二個是比較基準。它在那裡是為了讓你看到，並排比較你的技能實際帶來的差異——不只是「這個輸出還可以嗎」，而是「這個輸出比 Claude 自己做的更好嗎」。

審閱每一對並用白話文給出回饋。在審閱時，你只需回答兩個問題：
- 有技能的版本是我會使用的版本嗎？如果是，很好——記下讓它更好的原因，讓技能繼續這樣做
- 如果不是，缺少什麼或哪裡不對？具體一點。「語氣太正式」或「跳過了執行摘要」給 Claude 可以行動的依據；「不太對」則沒有

提交回饋後，Claude 根據你的意見修改技能。

### 迭代技能

每次修改一件事。如果第一輪顯示技能太冗長且缺少一個章節，選擇更重要的那個先修，再次執行，然後回來處理另一個。你能判斷什麼真正改變了。

大多數技能在一兩輪後就可以了。分享技能——給自己、給隊友——的門檻不是完美的評估結果。而是你關心的案例比基準明顯更好，以及你已知尚未處理的邊緣案例。

如果第一次輸出已經很好，你就完成了。eval 不是要跳過的關卡——它是在你需要信心時才用的，不是儀式。

## 14. 與團隊分享你建立的內容

### 在組織中分發外掛

在較大的公司中，分享外掛的推薦方式是通過組織的私有市集——一個管理員維護的公司核准外掛目錄。

分發是一次交接。你將外掛帶給負責你組織市集的人——可能是團隊主管、賦能或運營負責人，或 IT——他們在發布時選擇如何讓其他人接觸到它：

| 安裝等級 | 說明 |
|----------|------|
| Available（可用） | 出現在公司目錄中，有需要的人可以自行安裝。適合大多數團隊外掛 |
| Installed by default（預設安裝） | 隊友開啟 Cowork 時已在那裡，他們可以關閉 |
| Required（必要） | 安裝且保持開啟。保留給合規檢查或需要所有人以相同方式執行的工作 |
| Hidden（隱藏） | 在市集中但不顯示在目錄——用於分段或受限的推出 |

從隊友的角度，外掛只是在他們的目錄中出現，標記為來自公司，與 Anthropic 的公開外掛並排顯示。他們可以使用和關閉（除非是必要的），但不能編輯——更新由維護者推送。

### 保持外掛健康的習慣

- 具體命名技能和外掛：「meeting-prep」可能與組織外掛目錄中的其他三個會議準備技能衝突。「sales-customer-renewal-prep」則不會
- 設定定期回顧：每季度回顧已安裝的內容、實際被使用的內容，以及已過時的內容。淘汰沒人執行的，改進已發現改進機會的

### 反思

思考你和你身邊隊友目前在 Cowork 中最依賴的技能：
- 哪兩三個會是打包成團隊外掛的第一候選？
- 你組織中哪個人是你會將外掛帶去的人——能讓它上架到市集的人？

如果還不知道第二個問題的答案，那是你真正的下一步。

## 15. 課程總結與後續步驟

### 四個模組回顧

第一模組：認識 Claude Cowork
Cowork 是什麼、與 Chat 和 Code 的差異、適合的工作類型，以及第一個真實任務。

第二模組：讓 Cowork 為你所用
全域指令、專案、技能和外掛——當 Cowork 知道你知道的事、能存取你能存取的東西、以你的團隊方式做事時，它能變得多有能力。

第三模組：在任何地方使用 Claude
通過 Chrome 中的 Claude 進入瀏覽器，以及通過 Claude in M365 進入 Word、Excel、PowerPoint 和 Outlook。

第四模組：分享與安全
負責任地使用自主工具、驗證你建立的東西，以及將個人工作流程轉為團隊基礎設施。

貫穿全程的主線：Cowork 超越聊天，讓你委派真實工作。課程中的所有內容都建立在這個基礎之上。

### 本週建議行動（選一項）

- 設定記憶：撰寫一個五句話的全域指令，或為你目前進行中的一項工作流程建立一個專案
- 排程任務或設定 Dispatch：將一個週期性可交付成果設定為週期執行。或設定 Dispatch，讓下次在非工作時間有任何想法時，可以把它交給 Cowork
- 安裝一個外掛：前往「Customize → Plugins」，安裝最接近你角色的外掛，並針對本週的工作執行它
- 試用 Chrome 或 M365 中的 Claude：選擇與你大多數真實工作發生地最相符的介面，安裝它並用在一個任務上
- 分享你建立的東西：與你的團隊分享一個你覺得有價值的使用案例、一個產生了好結果的工作流程，或你建立的技能
